## 三、客户触达方案

### 3.1 线上渠道（低成本起步）

| 渠道 | 具体做法 | 预期效果 |
|------|---------|---------|
| **官网 SEO** | 关键词：跨境电商 AI 客服、Shopify 智能客服 | 长期自然流量 |
| **跨境电商社群** | 知乎/小红书/卖家论坛投稿案例 | 精准触达 |
| **YouTube/B站** | 录制产品 Demo 视频、使用教程 | 建立信任 |
| **独立站嵌入** | 在 Demo 站点加入 "Powered by Fiido" | 自然传播 |

### 3.2 精准触达（主动出击）

```
目标客户画像：
├── 行业：跨境电商独立站卖家
├── 平台：Shopify 店铺为主
├── 规模：月订单 500-10000 单
├── 痛点：客服成本高、夜间无人值守、多语言沟通
└── 地区：中国卖家出海 (UK/EU/US 市场)

触达方式：
├── 跨境电商展会 (Canton Fair、CCBE)
├── Shopify 合作伙伴生态
├── 卖家交流群/论坛定向推广
└── 老客户转介绍（给 10-20% 佣金）
```

### 3.3 官网必备要素

```
官网结构：
├── 首页 - 价值主张 + CTA (免费试用)
├── 产品介绍 - 功能详情 + Demo 视频
├── 定价页面 - 套餐对比 + 常见问题
├── 案例展示 - 客户故事 + 数据指标
├── 帮助中心 - 文档 + 视频教程
└── 联系我们 - 在线咨询 + 预约演示
```

## 五、支付与签约流程

### 5.1 收款方式对比

| 方式 | 适用阶段 | 是否需要公司 | 手续费 | 优缺点 |
|------|---------|-------------|-------|--------|
| **个人微信/支付宝** | 早期验证 | ❌ 不需要 | 0 | 简单但不专业，无法开票 |
| **个人收款码商家版** | 早期验证 | ❌ 不需要 | 0.38% | 支持信用卡，有流水限制 |
| **个体工商户** | 初期商业化 | ✅ 需要 | 0.38-0.6% | 可开普票，年收入 500 万以下 |
| **有限公司** | 规模化 | ✅ 需要 | 0.38-0.6% | 可开专票，融资需要 |
| **Stripe** | 海外客户 | ✅ 需要公司 | 2.9%+$0.30 | 国际支付必备 |

### 5.2 推荐路径

```
阶段一：验证期（0-10 个客户）
└── 个人微信/支付宝转账 + 微信聊天记录作为合同

阶段二：初期商业化（10-50 个客户）
└── 注册个体工商户 + 对公账户 + 电子合同

阶段三：规模化（50+ 客户）
└── 注册有限公司 + Stripe + 自动化计费系统
```

### 5.3 需要开发的支付功能

```python
# services/billing/service.py 需要实现：

class BillingService:
    # 套餐管理
    def get_plans() -> List[Plan]           # 获取套餐列表
    def create_subscription(plan_id)         # 创建订阅

    # 支付集成
    def create_payment_link(plan_id)         # 生成支付链接 (Stripe)
    def handle_webhook(event)                # 处理支付回调

    # 用量计费
    def check_quota(tenant_id, action)       # 检查配额
    def record_usage(tenant_id, usage)       # 记录用量
    def get_usage_stats(tenant_id)           # 用量统计

    # 账单管理
    def get_invoices(tenant_id)              # 获取账单
    def generate_invoice_pdf(invoice_id)     # 生成发票
```

### 5.4 电子合同

```
签约流程：
1. 客户在官网选择套餐 → 生成订单
2. 跳转 Stripe/支付宝完成付款
3. 支付成功 → 自动发送电子合同 (法大大/上上签)
4. 客户确认签署 → 账号自动开通
5. 发送欢迎邮件 + 使用指南
```

## 八、快速起步方案（半手动模式）

如果已有意向客户但系统还没完全准备好，可以采用半手动模式：

```
临时方案（无需开发）：
1. 通过微信/邮件沟通需求
2. 对公转账收款（或微信/支付宝转账）
3. 手动为客户配置 Shopify 连接
4. 手动创建坐席账号
5. 每月手动统计用量发送账单

优点：
- 零开发成本
- 快速验证产品市场匹配度 (PMF)
- 收集真实客户反馈

缺点：
- 人工成本高
- 无法规模化
- 客户体验一般
```
